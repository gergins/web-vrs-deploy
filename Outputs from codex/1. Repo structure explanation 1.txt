1. **Repo structure explanation**

Top-level directories and files:

- `.agents`
  - Contains Codex skill definitions under `.agents/skills/*`.
  - Subsystem: agent workflow/support.
  - Status: fairly complete for process guidance.

- `apps`
  - Contains deployable applications.
  - Subsystem: runtime implementation.
  - Status: core apps exist and are functional for prototype scope.

- `packages`
  - Shared packages: `contracts`, `shared`, `domain`, `ui`, `webrtc`.
  - Subsystem: shared code/contracts.
  - Status: mixed. `contracts` is strong; others are partial or placeholder.

- `infra`
  - Dockerfiles, nginx, TURN config, scripts, and a `k8s` folder.
  - Subsystem: local/deployment infrastructure.
  - Status: local/dev infra is present; production readiness is partial.

- `docs`
  - Architecture, flows, contracts, operations, specifications, verification, product notes.
  - Subsystem: authoritative repo truth.
  - Status: strong overall, but some areas remain partial/stub-like.

- `skills`
  - Human-readable debug/support notes.
  - Subsystem: support/process.
  - Status: support-only, not runtime.

- `tests`
  - `e2e`, `unit`, `integration`, `fixtures`.
  - Subsystem: verification.
  - Status: E2E has real tests; unit/integration are currently empty.

- `codex-vrs-engineering-templates`
  - Present as a top-level directory.
  - Subsystem: support/process/templates.
  - Status: exists, but not a core runtime area.

- Other top-level items
  - `AGENTS.md`: repo rules and truth hierarchy.
  - `project-manifest.json`: high-level product scope and protected semantics.
  - `docker-compose.yml`: local stack orchestration.
  - `.env`, `.env.example`: runtime and template env files.
  - `node_modules`, `.git`: normal workspace directories.
  - `.tmp-edge-cdp`, `*.log`, `repo*.txt/zip`: local artifacts/support noise, not authoritative.

4. **Recommended next steps**

Top 10 practical missing components for a stronger functional Web VRS system:

1. Add stable disconnect/reconnect regression coverage.
2. Implement worker-backed timeout/reaper behavior or explicitly remove that architecture from runtime expectations.
3. Implement real audit logging through `AuditLog`.
4. Implement real metrics/observability, even if minimal.
5. Complete disconnect/reconnect lifecycle hardening across all terminal paths.
6. Clarify and consolidate WebRTC ownership between `apps/web/src/webrtc` and `packages/webrtc`.
7. Add unit and integration tests for queue/session/signaling services.
8. Build a minimal real admin/operations view or clearly defer it from product scope.
9. Add a proper reporting service if reporting remains a declared backend domain.
10. If the target remains true VRS beyond browser-only prototype, begin telephony/SIP bridge planning and infrastructure, because that is currently absent.
